# W1.1 One-Page Offer Doc (V1 Build Process)

> Source: Execution Squad Mentoring - Week 1

## Core Philosophy

This is NOT "write the perfect offer."
This is "lock a testable v1."

**Nobody judges v1. Not you. Not me. The market does.**

You build this by brain dumping, then tightening, then locking.
Next week, you change it based on results. Not opinions.

---

## The Loop

```
Brain dump → GPT tightens → you answer → GPT tightens again → you lock v1
```

---

## Step 1: Brain Dump

Paste this into GPT. Answer only what you can. Skip anything you don't know.

### PROMPT 1: BRAIN DUMP (V1)

```
I'm building a v1 one-page offer doc. This is a hypothesis. Do not judge it.

Rules:
- I will answer only what I know.
- If I don't know, I will write "SKIP".
- After I answer, you must fill the missing parts with a reasonable example draft so I can react to it.
- Do not add new claims. Keep it plain and realistic.
- Keep everything short.

Questions (answer in short bullets):
- Who do I want to help?
- What is the main outcome they want? (one outcome)
- Where does it break today? (one leak)
- What do they lose because of that leak? (one sentence)
- What do I install or set up? (one sentence, no tool list)
- What are 1–4 deliverables I can confidently include? (bullets)
- What are 1–3 things I will NOT do? (bullets)
- What do I need from them to do this? (access/assets/time)
- What is the next step I want them to take? (one CTA)

Now do this in order:
A) Extract what I gave you into clean one-liners.
B) For every "SKIP", write a placeholder example that sounds like a real v1, marked clearly as: "[EXAMPLE PLACEHOLDER]".
C) Ask me only the top 3 questions that would improve the offer the most.
D) Output a draft of this structure (short, one page style):
   - one-line outcome
   - who it's for (2 lines)
   - the problem (1 line)
   - the outcome (1 line)
   - included (up to 4 bullets)
   - not included (up to 3 bullets)
   - how it works (3 steps)
   - what I need from them (bullets)
   - CTA (one line)
```

---

## Step 2: Tighten the Language

Paste this into GPT after Step 1.

### PROMPT 2: TIGHTEN

```
Take the draft and tighten it.

Rules:
- one idea per sentence
- no tool talk as the lead
- no extra promises
- keep it plain and direct
- keep one CTA only
- remove anything that sounds like marketing

Output two versions:
- Version A: more direct, sharper
- Version B: calmer, cleaner
```

---

## Step 3: Lock It for 7 Days

Paste this into GPT after you pick A or B.

### PROMPT 3: LOCK V1

```
This is v1. It is a hypothesis.

Turn it into a final one-page offer doc.

Rules:
- final must fit on one page
- one CTA only
- include a short "scope boundary" section
- include a short "what I need from you" section
- do not add new claims
- do not add new deliverables
- keep it ready to send today

Return the final one-page offer doc.
```

## Final Template

```
OFFER NAME
[Offer name]

ONE-LINE OUTCOME
I help [who] get [outcome] by installing [system] so [next step] happens without chaos.

WHO THIS IS FOR
You are [who].
You already have demand, but [leak].
You want [simple next step / consistency].

THE PROBLEM (one line)
Right now, [where it breaks]. That is why [desired outcome] is not happening consistently.

THE OUTCOME (one line)
After this, [what changes]. Simple. Trackable. Repeatable.

WHAT YOU GET (scope)
Included:
- [deliverable 1]
- [deliverable 2]
- [deliverable 3]
- [deliverable 4]

Not included:
- [not included 1]
- [not included 2]
- [not included 3]

HOW IT WORKS (plain)
Step 1: Decision Call. We confirm fit, scope, and the next action.
Step 2: Install. We connect it end-to-end so it runs.
Step 3: Run + Fix. We track what happens and fix the first leak.

WHAT I NEED FROM YOU
- [access needed 1]
- [access needed 2]
- [asset needed 1]
- [approval rule]

WHAT IT COSTS
[price]
[terms]

NEXT STEP (only CTA)
Book a decision call: [link]
```

## Application to NexOperandi

### Current State vs. One-Page Offer Philosophy

| Current Site | One-Page Offer Doc |
|--------------|-------------------|
| 3 solutions with 6+ features each | ONE outcome, ONE leak |
| 3 pricing tiers with 10 features | Clear scope (4 included, 3 not included) |
| Multiple CTAs per page | ONE CTA only |
| "We do everything" energy | "I solve THIS specific problem" |

## Key Insight

**V1 is a hypothesis, not a finished product.**

You're not trying to write the perfect offer - you're creating something testable so the *market* tells you what works, not your own opinions.

The website can stay as-is for now. The one-page offer doc is for **direct outreach/sales conversations**, not necessarily your marketing site (yet).
